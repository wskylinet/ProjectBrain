# Project Brain V1 - 产品需求文档（PRD）

## 1. 项目简介

### 项目名称

Project Brain

### 项目定位

面向研发、运维人员的项目档案管理平台。

用于统一管理公司多个地区项目的信息，包括：

* 项目基础信息
* VPN信息
* 服务器信息
* 数据库信息
* 部署信息
* 项目备注

解决当前项目资料分散、环境信息难查、交接困难等问题。

---

## 2. 项目背景

目前维护多个地区项目：

* 深圳
* 阜阳
* 泉州
* 柯桥
* 柯桥滨海
* 晋江
* 河科大
* 义乌
* 淮北

项目整体采用同一套代码进行部署，但不同地区存在：

* 不同VPN
* 不同服务器
* 不同数据库
* 不同部署方式
* 不同访问地址

目前这些信息主要依赖个人记忆或零散文档管理，查询和交接成本较高。

因此需要建设统一的项目档案管理平台。

---

## 3. MVP目标

V1目标：

建立统一项目档案中心。

实现：

* 项目统一管理
* 环境信息统一管理
* 项目资料统一存储

暂不实现：

* AI功能
* 需求池
* 运维经验库
* 知识搜索
* 权限系统
* 工作流

---

## 4. 功能模块

### 4.1 项目列表

#### 功能

展示所有项目信息。

#### 支持

* 查询
* 新增
* 编辑
* 删除
* 查看详情

#### 列表字段

| 字段   | 说明   |
| ---- | ---- |
| 项目名称 | 项目名称 |
| 地区   | 所属地区 |
| 系统名称 | 系统名称 |
| 负责人  | 负责人  |
| 创建时间 | 创建时间 |

### 4.2 项目详情

项目详情采用Tab方式展示。

---

#### Tab1：基础信息

字段：

* 项目名称
* 地区
* 系统名称
* 项目状态
* 负责人
* 联系方式
* 项目描述

---

#### Tab2：访问信息

字段：

* VPN类型
* VPN名称
* VPN说明
* 测试地址
* 生产地址
* 文档地址

说明：

密码不存储。

仅存储使用说明。

---

#### Tab3：服务器信息

支持维护多台服务器。

字段：

* 服务器名称
* 服务器IP
* 远程端口
* 服务器用途
* 操作系统
* 备注

用途示例：

* Web服务器
* 数据库服务器
* Redis服务器
* 文件服务器

---

#### Tab4：数据库信息

支持维护多个数据库。

字段：

* 数据库类型
* 数据库地址
* 数据库名称
* 备注

数据库类型：

* SQL Server
* MySQL
* PostgreSQL
* SQLite
* Redis
* InfluxDB
* TDengine

---

#### Tab5：部署信息

字段：

* IIS站点名称
* 部署目录
* 日志目录
* 发布方式
* 备注

发布方式：

* IIS
* Docker
* Windows服务

---

#### Tab6：附件资料

支持上传：

* Word
* Excel
* PDF
* 图片

用途：

* 部署文档
* 环境说明
* 项目截图
* 客户资料

---

#### Tab7：备注

自由文本记录。

例如：

* 需先连接EasyConnect
* 使用堡垒机访问
* 数据同步服务部署在独立服务器
